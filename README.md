**RiftMart** is an AI-native e-commerce experience built for autonomous agents, not human shoppers ‚ÄĒ
wrapped in a retro Y2K / Windows-XP-era terminal that makes the AI's shopping mission feel like
you're hacking into a futuristic OS.

## ūüďĖ Overview

Traditional e-commerce is built for humans clicking through pages. **RiftMart flips that** ‚ÄĒ it's a
shopping platform designed for an AI agent to operate end-to-end. You give it two things, a
**product** and a **budget**, and the agent takes it from there: searching, comparing prices,
reading reviews, picking a winner, checking out, and paying ‚ÄĒ while you watch the entire mission
play out inside a terminal that looks like it booted off a 2004 BIOS chip.

No dashboards. No product grids. Just a black screen, a blinking cursor, and an AI doing your
shopping for you.

## ‚ú® Features

- ūü§Ė **Autonomous AI shopping agent** ‚ÄĒ takes a product + budget and runs the whole mission unattended
- ūüĖ•ÔłŹ **AgentOS Terminal** ‚ÄĒ a fully custom, from-scratch terminal UI (no `cmd.exe`/xterm dependency)
- ūüíĺ **BIOS-style boot sequence** ‚ÄĒ memory check, kernel load, driver init, animated progress bar
- ūüí¨ **Conversational AI intake** ‚ÄĒ natural back-and-forth to capture what to buy and how much to spend
- ūüĒć **Simulated multi-source search** ‚ÄĒ Flipkart / Amazon comparison, price comparison, review checks
- ūüõí **Shop.exe** ‚ÄĒ the actual product catalog, search, and checkout flow the agent drives
- ūüíį **AI-Pay Protocol** ‚ÄĒ a mock payment rail with rule-based approval and transaction records
- ūüßĺ **ASCII receipt** ‚ÄĒ a printable, copyable transaction summary at the end of every mission
- ūüé® **Y2K / retro-CRT aesthetic** ‚ÄĒ scanlines, CRT flicker, glow text, and a fully custom color system
- ‚ö° **Modular architecture** ‚ÄĒ terminal, shop, agent, and payment are independent services with clean interfaces between them

## ūüß≠ How It Works

The user only ever provides **two inputs** ‚ÄĒ everything else is autonomous.

```mermaid
flowchart TD
    U["ūüßĎ User\n(product + budget)"] --> T[AgentOS Terminal]
    T --> C[AI Conversation]
    C --> A[Shopping Agent]
    A --> S["Shop.exe\n(search ¬∑ compare ¬∑ evaluate)"]
    S --> Cart[Add to Cart]
    Cart --> Checkout[Checkout]
    Checkout --> P["AI-Pay Protocol\n(approve / reject)"]
    P --> R["ūüßĺ Receipt\nMISSION COMPLETE"]
```

### The terminal experience, phase by phase

| # | Phase | What happens |
|---|-------|--------------|
| 1 | **Boot Screen** | RIFT BIOS boots up ‚ÄĒ memory check, kernel + driver loading, progress bar to 100% |
| 2 | **ASCII Logo** | The RIFT MART wordmark glitches in and settles into a steady CRT glow |
| 3 | **Terminal** | A live `RIFT>` command line ‚ÄĒ real keyboard input, command history, built-in commands (`help`, `buy`, `clear`, `about`...) |
| 4 | **AI Conversation** | The agent asks what to buy and what your budget is, then runs a simulated multi-source search |
| 5 | **Shop.exe Launch** | Control hands off to the shopping engine, which searches, compares, and evaluates products |
| 6 | **Purchase Complete** | The agent reports back the product it picked, the price, and the seller |
| 7 | **Receipt** | An ASCII transaction receipt ‚ÄĒ savable, copyable |
| 8 | **Shutdown** | Session teardown, memory cleared, power off |

No manual clicking required after the mission starts ‚ÄĒ the agent drives the entire flow from
search to payment.

## ūüöÄ Getting Started

### Prerequisites

- **Node.js** 18+ and npm
- **Python** 3.10+
- **Git**

### Clone the repo

```bash
git clone https://github.com/shashwat230710/Rift-Mart.git
cd Rift-Mart
```

### 1. Frontend (AgentOS Terminal)

```bash
cd frontend
npm install
npm run dev
```

Runs at `http://localhost:5173` by default.

### 2. Backend (Shop.exe API)

```bash
cd backend
pip install -r requirements.txt --break-system-packages
uvicorn main:app --reload
```

> Adjust the module name in the `uvicorn` command if your entrypoint file isn't `main.py`.

### 3. Agent

```bash
cd agent
pip install -r requirements.txt --break-system-packages
python agent.py
```

The agent is designed to run locally alongside the frontend/backend ‚ÄĒ it isn't meant to be hosted.

### 4. Payment (AI-Pay Protocol)

```bash
cd payment
pip install -r requirements.txt --break-system-packages
uvicorn main:app --reload --port 8001
```

Once all four are running, open the frontend URL and start a mission from the terminal
(`buy <product>` once you're past the boot/logo screens).

## ūüéģ Usage

1. Wait for (or skip past) the BIOS boot sequence and logo
2. At the `RIFT>` prompt, type `help` to see available commands
3. Type `buy <product name>` ‚ÄĒ e.g. `buy gaming mouse`
4. Answer the AI's budget question
5. Watch the agent search, compare, and check out automatically
6. Read your ASCII receipt at the end ‚ÄĒ `MISSION COMPLETE`

## ūüé® Theme

RiftMart deliberately avoids looking like a normal website. The whole interface is built around:

- CRT scanlines, screen flicker, and vignette
- A DOS/BIOS-inspired boot sequence rather than a login page
- Glitch-in text reveals and blinking block cursors instead of fades
- A constrained retro color palette (green, amber, white, error red, and a touch of blue)

The goal: **don't build a fake operating system ‚ÄĒ build the illusion of one.**
